
# 优化与调试建议

## 时序校准

如果 LED 显示颜色不正确或闪烁，请校准 `ws2812_send_byte_simple()` 中的 NOP 数量：

**现象诊断**:
- 全白偏蓝 → 高电平太长 (1码太长)
- 全白偏绿 → 高电平太短 (0码太长)
- 闪烁、乱码 → 时序偏差太大

**用示波器测试**:
测量 RB1 引脚波形:
- T0H (0码高电平): 300-400ns
- T0L (0码低电平): 800-900ns
- T1H (1码高电平): 700-900ns
- T1L (1码低电平): 300-400ns

## 汇编版驱动

`ws2812_send_byte()` 是精确汇编版本，但需要调试。如果简单版工作正常，优先使用简单版。

## 2.5V 低电压注意事项

SC8F072 在 2.5V 可以工作，但需要注意:

1. **频率选择**: 建议使用 4MHz (而非 16MHz)
   ```c
   #define USE_16MHZ  0
   ```

2. **WS2812B 电平兼容性**:
   - WS2812B VIH = 0.7 * VDD
   - 如果 LED 5V 供电, VIH = 3.5V
   - 2.5V 输出高电平无法达到 3.5V
   - **解决方案**: 加 MOS 管电平转换或使用 3.3V 供电的 LED

3. **LVR 设置**:
   配置字中 LVR 设为 2.0V，防止电压波动复位

## 低功耗优化

1. 不发送数据时进入 SLEEP
2. 关闭未使用的外设 (ADC, PWM)
3. 未使用引脚设为输出低或开上拉

## 常见问题排查

**问题 1: LED 不亮**
1. 检查 RB1 是否设为输出: TRISB & (1<<1) == 0
2. 检查 ANSEL1 是否清零: ANSEL1 & (1<<1) == 0
3. 检查电源和接线
4. 上电后应该有白色指示灯效果

**问题 2: LED 颜色不对**
1. WS2812B 是 GRB 顺序，不是 RGB!
2. 检查时序: 增加或减少 NOP
3. 确认是否开启了中断干扰时序

**问题 3: 按键无效**
1. 检查 RA0 上拉是否开启: WPUA & (1<<0)
2. 检查 RA0 是否设为输入: TRISA & (1<<0)
3. 检查是否开启了全局中断 GIE=1

**问题 4: 程序反复复位**
1. WDT 是否意外开启? (OSCCON bit1)
2. LVR 阈值是否设置太高?
3. 电源电压是否稳定

## 烧录测试步骤

1. **第一阶段**: 只烧简单的 IO 测试
   ```c
   // 让 RB1 输出 1Hz 方波
   void main() {
       TRISB &amp;= ~(1&lt;&lt;1);
       while(1) {
           PORTB ^= (1&lt;&lt;1);
           __delay_ms(500);
       }
   }
   ```

2. **第二阶段**: 测试 WS2812B 发送固定颜色
   ```c
   // 发送红色
   RGB_t red = {0, 100, 0};  // G=0, R=100, B=0
   ws2812_send_pixel(red);
   ws2812_reset();
   ```

3. **第三阶段**: 完整功能测试

## WS2812B 数据格式

```
  发送顺序: G7 G6 ... G0, R7 R6 ... R0, B7 B6 ... B0

  第一个像素的第一个字节: 绿色 (高位先发)
```

常见错误: 把 RGB 顺序搞错了！
