# BUG修复工作流程

## 必须遵循的步骤（每次修复都必须完成）

### 第1步：查看错误信息
```
Read output/cmscerr.err
记录：错误文件、错误行号、错误类型
```

### 第2步：读取相关代码
```
读取错误行附近的代码（前后各20行）
读取相关的头文件
检查依赖关系
```

### 第3步：分析问题
```
确定根本原因
检查是否有类似问题
考虑修复方案的影响
```

### 第4步：进行修复
```
修改代码
确保修改最小化
保持代码风格一致
```

### 第5步：自行检查两遍
```
检查1：验证语法正确性
- 读取修改后的代码
- 检查是否有明显的语法错误
- 检查是否有遗漏的括号、分号

检查2：验证逻辑正确性
- 检查是否会影响其他代码
- 检查是否有潜在的副作用
- 验证修复确实解决了问题
```

## 检查清单

修改前：
- [ ] 已查看错误日志
- [ ] 已读取相关代码
- [ ] 已分析问题原因

修改后：
- [ ] 已进行第一遍检查
- [ ] 已进行第二遍检查
- [ ] 确认不会引入新问题
