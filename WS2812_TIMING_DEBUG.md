# WS2812 时序调试指南

## 问题分析

当前WS2812颜色显示不正常，通常是由于时序不符合WS2812的要求导致的。

### WS2812标准时序要求

| 数据位 | 高电平时间 | 低电平时间 | 总周期 |
|--------|-----------|-----------|--------|
| 1码    | 700-900ns | 200-400ns | ~1250ns |
| 0码    | 200-400ns | 600-800ns | ~1250ns |

### 当前配置（16MHz，2T模式）

- 1个NOP = 125ns
- 1码: 高6NOP(750ns) + 低4NOP(500ns) = 1250ns ❌ 低电平偏长
- 0码: 高4NOP(500ns) + 低6NOP(750ns) = 1250ns ❌ 高电平偏长

**问题：**当前实现中，1码的低电平和0码的高电平时间超出了WS2812的要求范围。

## 调试步骤

### 方法1：使用简单测试程序（推荐）

文件：`ws2812_simple_test.c`

1. 打开 `ws2812_simple_test.c`
2. 修改 `TIMING_CONFIG` 的值来调整时序
3. 选择测试模式（推荐先用 `COLOR_CYCLE`）
4. 编译并烧录
5. 观察LED颜色是否正确

**推荐时序配置：**

```c
// 配置1: 标准时序
#define TIMING_CONFIG 6, 2, 2, 6

// 配置2: 1码高电平更长
#define TIMING_CONFIG 7, 2, 2, 6

// 配置3: 保守配置
#define TIMING_CONFIG 7, 3, 3, 7
```

### 方法2：使用示波器/逻辑分析仪

文件：`test_ws2812_timing.c`

1. 将测试模式设置为 `TIMING_TEST`
2. 连接示波器到 RB1 引脚
3. 观察以下内容：
   - 1码的高电平时间（应在700-900ns）
   - 1码的低电平时间（应在200-400ns）
   - 0码的高电平时间（应在200-400ns）
   - 0码的低电平时间（应在600-800ns）

### 方法3：手动修改ws2812_send_byte函数

直接修改 `ws2812.c` 中的 `ws2812_send_byte` 函数：

```c
void ws2812_send_byte(uint8_t data) {
    uint8_t i;
    for (i = 0; i < 8; i++) {
        if (data & 0x80) {
            // 发送1码 - 调整NOP数量
            RB1 = 1;
            asm("nop"); asm("nop"); asm("nop"); asm("nop"); asm("nop"); asm("nop"); // 6个NOP = 750ns
            RB1 = 0;
            asm("nop"); asm("nop"); // 2个NOP = 250ns
        } else {
            // 发送0码 - 调整NOP数量
            RB1 = 1;
            asm("nop"); asm("nop"); // 2个NOP = 250ns
            RB1 = 0;
            asm("nop"); asm("nop"); asm("nop"); asm("nop"); asm("nop"); asm("nop"); // 6个NOP = 750ns
        }
        data <<= 1;
    }
}
```

## 常见时序问题及解决方案

### 症状：LED完全不亮

**可能原因：**
- 时序完全错误
- 数据引脚配置错误
- 复位时间不够

**解决方案：**
- 检查TRISB1是否设置为输出
- 增加复位延时（当前60us，WS2812要求>50us）
- 用示波器确认数据线有信号输出

### 症状：LED亮但颜色错误

**可能原因：**
- 时序接近但不精确
- 高低电平时间比例不对

**解决方案：**
- 尝试不同的时序配置
- 使用示波器测量实际时序
- 检查电源电压是否稳定（4.5-5.5V）

### 症状：颜色不稳定或闪烁

**可能原因：**
- 时序不稳定
- 中断干扰
- 电源问题

**解决方案：**
- 发送数据时禁用中断
- 检查电源去耦电容
- 确保数据线传输稳定

## 颜色测试标准

测试时应该看到：

| 设置值 | 颜色 |
|--------|------|
| (255, 0, 0) | 红色 |
| (0, 255, 0) | 绿色 |
| (0, 0, 255) | 蓝色 |
| (255, 255, 255) | 白色 |
| (255, 0, 255) | 品红 |
| (255, 255, 0) | 黄色 |
| (0, 255, 255) | 青色 |

**注意：** WS2812使用GRB格式，而非RGB格式！

## 推荐的调试流程

1. 首先使用 `ws2812_simple_test.c` 的 `TIMING_TEST` 模式
2. 用示波器观察RB1引脚，测量实际时序
3. 调整NOP数量使时序符合WS2812要求
4. 切换到 `COLOR_CYCLE` 模式测试颜色
5. 如颜色仍有偏差，微调时序参数
